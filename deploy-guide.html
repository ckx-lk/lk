1
注册 Vercel 并导入项目
1.1 注册 Vercel 账号
  • 打开 vercel.com
  • 点击右上角 Sign Up
  • 选择 Continue with GitHub，用 ckx-lk 账号登录
  • 授权 Vercel 访问你的 GitHub 仓库
1.2 导入 lk 仓库
  • 登录后点击 Add New → Project
  • 在 Import Git Repository 中找到 ckx-lk/lk 仓库
  • 点击 Import
💡 提示：如果看不到 lk 仓库，点击下方的 "Adjust GitHub App Permissions"，授权 Vercel 访问 ckx-lk 组织的仓库。
1.3 配置并部署
  • Framework Preset 保持 Other 即可
  • Root Directory 设置为 /（默认）
  • Build Command 和 Output Directory 留空（纯静态站点无需构建）
  • 点击 Deploy 按钮
✅ 部署成功后，Vercel 会分配一个地址如 lk-xxxx.vercel.app，点击即可预览网站。
2
在 Vercel 中绑定域名
2.1 添加域名
  • 进入项目 → Settings → Domains
  • 输入 www.lankai.net，点击 Add
  • 再输入 lankai.net，点击 Add
💡 提示：同时添加 www 和裸域名，Vercel 会自动将 lankai.net 重定向到 www.lankai.net。
2.2 查看 Vercel 提供的 DNS 记录

添加域名后，Vercel 会显示需要配置的 DNS 记录。页面会类似如下：

类型 主机记录 记录值
CNAME www cname.vercel-dns.com
A @ 76.76.21.21
⚠️ 重要：以 Vercel 页面实际显示的值为准！如果 Vercel 给出了不同的 CNAME 目标地址，请使用 Vercel 显示的值。
3
在阿里云配置 DNS 解析
3.1 进入阿里云 DNS 管理页面
  • 登录 阿里云控制台（aliyun.com）
  • 搜索 "云解析 DNS" 或直接访问 dns.console.aliyun.com
  • 在域名列表中找到 lankai.net，点击 解析设置
3.2 添加 CNAME 记录（www 子域名）

点击"添加记录"，填写以下信息：

字段值
记录类型CNAME
主机记录www
解析路线默认
记录值cname.vercel-dns.com
TTL10分钟（600）

点击 确认 保存。

3.3 添加 A 记录（裸域名）

再次点击"添加记录"，填写以下信息：

字段值
记录类型A
主机记录@
解析路线默认
记录值76.76.21.21
TTL10分钟（600）

点击 确认 保存。

⚠️ 冲突检查：如果 @ 主机记录下已存在其他 A 记录（如旧网站指向的 IP），需要先删除旧记录，否则新记录无法添加或不会生效。
3.4 确认解析记录

配置完成后，DNS 解析列表应显示如下两条记录：

记录类型主机记录记录值状态
CNAMEwwwcname.vercel-dns.com✅ 正常
A@76.76.21.21✅ 正常
4
验证与生效
4.1 回到 Vercel 确认域名状态
  • 返回 Vercel → Settings → Domains
  • 等待 1-5 分钟，域名状态应从 "Invalid Configuration" 变为 "Valid Configuration" ✓
💡 如果显示 "Invalid"：DNS 传播需要时间，最长可能等待数小时。可用下方命令检查 DNS 是否已生效。
4.2 验证 DNS 生效

在终端运行以下命令检查：

dig www.lankai.net
应能看到 www.lankai.net 指向 cname.vercel-dns.com

或者在浏览器直接访问：

  • www.lankai.net — 应显示兰凯律所网站首页
  • lankai.net — 应自动跳转到 www.lankai.net
4.3 配置 SSL 证书

Vercel 自动为绑定的域名配置免费 SSL 证书，无需额外操作。DNS 生效后，访问 https://www.lankai.net 即可看到安全锁标识。
